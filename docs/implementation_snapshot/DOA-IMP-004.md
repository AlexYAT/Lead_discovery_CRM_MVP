журнал коммуникаций для Lead с добавлением попыток контакта и отображением истории в карточке лида.

- сохранена привязка по `lead_id` к существующей схеме SQLite;
- обеспечена хронология отображения по `date DESC, id DESC`.

# Как работает contact log
- В карточке лида добавлена SSR-форма `POST /leads/{lead_id}/contacts` с полями:
  - `date`
  - `message_text`
  - `outcome`
  - `next_action`
- На отправке форма создает запись в `contact_attempts`.
- После сохранения выполняется redirect обратно в карточку лида.
- В карточке отображается блок "История контактов" со всеми попытками по лидy.

# Как данные связаны с Lead
- Связь используется через `contact_attempts.lead_id -> leads.id` (существующий foreign key).
- При добавлении попытки используется `lead_id` из URL карточки.
- В выборке истории применяется фильтр `WHERE lead_id = ?`.
- Проверка сценария подтвердила, что попытки сохраняются и читаются только для соответствующего лида.
