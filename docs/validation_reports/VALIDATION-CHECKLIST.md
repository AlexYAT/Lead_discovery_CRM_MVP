# Validation Checklist

Назначение: воспроизводимая проверка MVP человеком в рамках validation-cycle.

Ограничения выполнения:
- фиксируются только наблюдения/проблемы;
- запрещены предложения решений;
- запрещены продуктовые изменения в ходе проверки.

## VAL-01 — Создание лида
- Цель проверки: подтвердить, что человек может вручную создать новый lead.
- Шаги:
  1. Открыть страницу `/leads`.
  2. Заполнить обязательные поля формы создания lead.
  3. Нажать кнопку создания.
- Expected observable outcome:
  - система создаёт lead;
  - выполняется переход в карточку созданного lead;
  - новый lead присутствует в списке.

## VAL-02 — Список лидов
- Цель проверки: подтвердить доступность и читаемость списка lead.
- Шаги:
  1. Открыть страницу `/leads`.
  2. Просмотреть отображаемые записи.
  3. Проверить переход в карточку по выбранному lead.
- Expected observable outcome:
  - список открывается без ошибок;
  - отображаются ключевые поля lead;
  - переход в карточку работает.

## VAL-03 — Карточка лида
- Цель проверки: подтвердить доступность полной карточки lead.
- Шаги:
  1. Из списка открыть карточку lead.
  2. Просмотреть профильные поля и текущий статус.
  3. Проверить отображение секций карточки.
- Expected observable outcome:
  - карточка открывается без ошибок;
  - профильные данные отображаются;
  - секции карточки доступны человеку.

## VAL-04 — Изменение статуса
- Цель проверки: подтвердить возможность изменения статуса lead через интерфейс.
- Шаги:
  1. В карточке lead открыть секцию обновления статуса.
  2. Выбрать доступный статус.
  3. Сохранить изменение.
- Expected observable outcome:
  - статус обновляется;
  - обновлённый статус отображается в карточке;
  - при необходимости изменение видно в списке лидов.

## VAL-05 — Заметки
- Цель проверки: подтвердить сохранение и повторное отображение заметок lead.
- Шаги:
  1. В карточке lead открыть секцию заметок.
  2. Ввести/изменить текст заметки.
  3. Сохранить заметку.
- Expected observable outcome:
  - заметка сохраняется;
  - после обновления страницы текст заметки не теряется.

## VAL-06 — Contact Log
- Цель проверки: подтвердить фиксацию попыток контакта и отображение истории.
- Шаги:
  1. В карточке lead открыть секцию Contact Log.
  2. Добавить новую попытку контакта.
  3. Проверить блок истории контактов.
- Expected observable outcome:
  - попытка контакта сохраняется;
  - запись появляется в истории контактов этого lead.

## VAL-07 — Consultation
- Цель проверки: подтвердить управление консультациями в карточке lead.
- Шаги:
  1. В карточке lead создать consultation.
  2. Проверить появление consultation в списке.
  3. Изменить status/result consultation и сохранить.
- Expected observable outcome:
  - consultation создаётся и отображается;
  - изменения status/result сохраняются и отображаются.

## VAL-08 — Dashboard
- Цель проверки: подтвердить доступность и базовую корректность агрегатов dashboard.
- Шаги:
  1. Открыть страницу `/dashboard`.
  2. Проверить общее количество lead.
  3. Проверить распределение по статусам.
- Expected observable outcome:
  - dashboard открывается без ошибок;
  - агрегаты отображаются;
  - данные выглядят согласованными с текущим состоянием системы.
