# Summary
Фиксация validation-цикла для MVP Lead Discovery CRM.

# Тип цикла
Human validation & UX feedback cycle

# Контекст
MVP полностью реализован и зафиксирован (`DOA-IMP-007`).
Система функционально работает, но не проходила проверку реальным пользователем.

# Цель
Проверить систему глазами пользователя и выявить UX/flow проблемы.

# Жёсткое ограничение
Запрещено:
- предлагать улучшения;
- менять архитектуру;
- реализовывать новые функции.

Разрешено только:
- наблюдение;
- фиксация проблем.

# Сценарии использования
- создание лида;
- просмотр списка лидов;
- работа с карточкой лида;
- добавление contact attempts;
- создание и обновление consultation;
- просмотр dashboard.

# Что считается проблемой
- UX-трение (лишние действия, непонятные шаги);
- неочевидная логика;
- ошибки интерфейса;
- несоответствие ожиданиям пользователя;
- неудобные или лишние поля;
- разрывы в flow.

# Формат фиксации проблем
- где возникает (экран / действие);
- описание проблемы;
- наблюдаемое поведение;
- ожидаемое поведение (без решения).

# Definition of Done
- пройдены ключевые пользовательские сценарии;
- зафиксирован список проблем;
- нет предложений решений;
- готово к переходу в ARCH/DEC следующего цикла.
