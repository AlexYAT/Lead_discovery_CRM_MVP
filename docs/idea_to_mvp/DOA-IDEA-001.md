# IDEA: Lead Discovery CRM - MVP

## Summary
Этот документ фиксирует исходную идею MVP для проекта Lead Discovery CRM на этапе жизненного цикла IDEA.
Основание идеи: предоставленный source input "Lead Discovery CRM - MVP Project Schema".
Цель документа - зафиксировать проблему, границы, ключевые сущности и критерии успеха до перехода к ARCH.

## Goal
Сформировать понятную и проверяемую основу MVP, которая позволяет:
- находить потенциальных клиентов;
- отбирать релевантных лидов;
- вести ручную коммуникацию;
- фиксировать итоги взаимодействий;
- отслеживать состояние воронки по стадиям.

## MVP Scope
### In scope
- Lead CRM;
- ручное создание лида;
- логирование контактов;
- управление статусами лидов;
- базовый dashboard с метриками.

### Out of scope
- автоматические сообщения;
- AI sales agent;
- сложные интеграции с внешними системами.

## Core Idea
MVP строится как минимальная CRM-система для ручного lead discovery и ручного sales-процесса.
Ключевая ценность - не автоматизация коммуникации, а структурированное ведение лидов и измеряемый pipeline.

Проблема и контекст:
- потенциальные клиенты находятся в разных источниках и быстро теряются без единого реестра;
- коммуникации ведутся вручную и без стандартизированного журнала взаимодействий;
- отсутствует прозрачность по текущему этапу воронки и результатам контактов;
- невозможно стабильно оценить конверсию без системной фиксации статусов и исходов.

## User Flow
1. Пользователь вручную добавляет лида.
2. Пользователь просматривает карточку лида и уточняет данные.
3. Пользователь вручную связывается с лидом.
4. Пользователь логирует попытку контакта и результат.
5. Пользователь отслеживает дальнейший статус и итог по лидy.

## Status Model
Базовая статусная модель лида:
- new
- reviewed
- contacted
- replied
- consultation_booked
- converted
- not_interested

Назначение модели:
- обеспечить единый язык стадий;
- сделать воронку наблюдаемой;
- упростить расчет конверсии между этапами.

## Success Criteria
### Definition of Done (MVP)
MVP считается завершенным, если:
1. Лид может быть создан вручную.
2. Доступен список лидов.
3. Реализована карточка лида.
4. Статус лида можно обновлять.
5. Можно зафиксировать попытку контакта.
6. История взаимодействий видима.
7. Dashboard отображает ключевые метрики.
8. Данные сохраняются после перезапуска.

### Acceptance Criteria
Пользователь (заказчик) может:
- добавлять лидов;
- контактировать с лидами;
- отслеживать ответы;
- управлять консультациями;
- видеть воронку и динамику по статусам.

## Constraints
- Этап жизненного цикла: IDEA (без перехода к ARCH/DEC/OP/IMP в рамках данного документа).
- Подход docs-first и create-only.
- Режим MVP: минимальная сложность, markdown-first, ручная проверка, один пользователь.
- Без автоматизации коммуникаций и без сложных интеграций.
