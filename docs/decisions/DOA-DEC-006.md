## Context

- CRM-ядро реализовано (Candidate Queue → Lead → CRM lifecycle)
- отсутствует источник входящих лидов
- требуется механизм обнаружения пользователей с выраженными pain signals
- приоритет проекта:
  качество лидов > стоимость > скорость
- pipeline предполагает редкий запуск (debug / периодический режим), а не постоянный поток

---

## Decision

Выбрана стратегия:

**Search Engine + LLM-based classification**

---

## Strategy Description

Концептуальная модель:

- внешний search-слой используется для первичного обнаружения кандидатов (recall)
- search выполняет базовую фильтрацию и ранжирование контента
- полученные кандидаты проходят через интеллектуальную классификацию (precision)
- классификация направлена на выявление pain signals
- результатом является формирование кандидатов для Candidate Queue

---

## Rationale

- search-движок уже решает задачу первичного отбора и ранжирования
- LLM обеспечивает высокую точность определения проблемных состояний
- стратегия минимизирует шум по сравнению с keyword-подходом
- соответствует приоритету качества лидов
- устраняет необходимость сложной эвристической системы
- лучше подходит для редких запусков, чем для потокового ingestion

---

## Alternatives Considered

### Keyword-based

- высокий уровень шума
- недостаточное качество кандидатов

### Keyword + Heuristic

- сложность настройки
- нестабильное качество
- требует постоянной ручной калибровки

### LLM-only

- отсутствует механизм первичного discovery
- не решает задачу поиска кандидатов

---

## Trade-offs

- увеличение стоимости обработки
- увеличение latency
- зависимость от внешнего search-слоя
- меньшая предсказуемость результата по сравнению с rule-based подходами

---

## Constraints

- используются только публичные данные
- соблюдаются правила платформ
- отсутствует агрессивный scraping
- отсутствует автоматическая коммуникация с пользователями

---

## Result

- стратегия discovery определена
- устранена неопределённость перед ARCH
- зафиксирован фокус на качестве лидов
- система готова к архитектурному проектированию
